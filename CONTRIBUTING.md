## 🚀 Як почати

### 1. Форкніть репозиторій

Натисніть кнопку **Fork** в правому верхньому кутку сторінки GitHub.

### 2. Клонуйте ваш fork

```bash
git clone https://github.com/YOUR-USERNAME/ScreenGuard.git
cd ScreenGuard
```

### 3. Додайте upstream

```bash
git remote add upstream https://github.com/ORIGINAL-OWNER/ScreenGuard.git
```

### 4. Створіть нову гілку

```bash
git checkout -b feature/your-feature-name
# або
git checkout -b fix/bug-name
```

## 💻 Розробка

### Встановлення середовища

```bash
# Переконайтесь, що маєте Android Studio та JDK 11+
java -version

# Синхронізуйте Gradle
./gradlew sync
```

### Запуск тестів

```bash
# Запустіть всі тести
./gradlew test

# Запустіть на пристрої
./gradlew installDebug

# Запустіть конкретний тест
./gradlew test --tests ClassName
```

### Код-стиль

Цей проект слідує **Kotlin Coding Conventions**:
- Використовуйте 4 пробіли для відступу
- Макс 120 символів на рядок
- Імпортуйте тільки потрібні класи
- Документуйте публічні функції

### Приклад

```kotlin
/**
 * Описує функцію
 * @param param1 Опис параметра
 * @return Опис повернення
 */
fun myFunction(param1: String): String {
    return param1.uppercase()
}
```

## 📝 Комітування

### Повідомлення про коміти

Слідуйте цьому форматові:

```
[TYPE] Brief description

Detailed explanation if needed.

Fixes #123
```

### Типи комітів

- `[feat]` - Нова функція
- `[fix]` - Виправлення баги
- `[docs]` - Документація
- `[style]` - Форматування, без змін коду
- `[refactor]` - Переорганізація коду
- `[test]` - Додавання/виправлення тестів
- `[chore]` - Оновлення залежностей

### Приклади

```bash
git commit -m "[feat] Add face recognition whitelist"
git commit -m "[fix] Fix notification crash on Android 13"
git commit -m "[docs] Update README with setup guide"
```

## 🔄 Pull Request

### Перед тим як відправити PR

- [ ] Коммітьте часто
- [ ] Пишіть змістовні повідомлення про коміти
- [ ] Синхронізуйте з upstream
- [ ] Тестуйте локально
- [ ] Переконайтесь, що Gradle build успішний

### Синхронізація з upstream

```bash
git fetch upstream
git rebase upstream/main
# або
git merge upstream/main
```

### Завантаження на fork

```bash
git push origin feature/your-feature-name
```

### Створення Pull Request

1. Перейдіть на GitHub
2. Натисніть "Compare & pull request"
3. Заповніть PR форму:

```markdown
## Опис

Коротко опишіть зміни.

## Тип змін

- [ ] 🐛 Виправлення баги
- [ ] ✨ Нова функція
- [ ] 📚 Документація
- [ ] ♻️ Рефакторинг

## Як тестувати

Дайте кроки для тестування:

1. ...
2. ...

## Чек-лист

- [ ] Мій код слідує код-стилю проекту
- [ ] Я запустив тести локально
- [ ] Я оновив документацію
- [ ] Нема breaking changes
```

## 🐛 Повідомлення про баги

### Використовуйте GitHub Issues

Коли знайдете баг, створіть новий issue з:

1. **Назва**: Коротке, чітке описання
2. **Опис**: Детальний опис проблеми
3. **Воспроизведення**: Кроки для відтворення
4. **Очікуваний результат**: Що мало б бути
5. **Фактичний результат**: Що насправді відбулось
6. **Інформація про середовище**:
   - Android версія
   - Пристрій/емулятор
   - Версія ScreenGuard

### Приклад

```markdown
**Описання**
Додаток падає при включенні захисту на Android 12

**Кроки для відтворення**
1. Відкрийте додаток
2. Натисніть "Активувати захист"
3. Дайте дозвіл на камеру
4. Додаток падає

**Очікуваний результат**
Додаток повинен запустити детекцію облич

**Фактичний результат**
com.screenguard.protector E/AndroidRuntime: FATAL EXCEPTION

**Інформація**
- Android: 12
- Пристрій: Pixel 5
- ScreenGuard: 1.0.0
```
