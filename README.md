# ScreenGuard - Android Security Application

**Версія**: 1.1.0 (🆕 НОВИЙ!)  
**Статус**: 🟢 Готово до production  
**Останнє оновлення**: Квітень 14, 2026

Сучасний додаток для Android, який захищає вашу приватність від небажаних спостерігачів. Коли хтось дивиться на ваш телефон без дозволу, селфі-камера активується, записує знімок та видає попередження!

## 🎯 Основні функції

### 1. **Детекція осіб у реальному часі**
   - Використовує ML Kit Face Detection від Google
   - Аналізує зображення з селфі-камери постійно
   - Миттєво виявляє наявність осіб на екрані
   - ⚡ Оптимізовано для швидкої роботи

### 2. **🆕 Автоматичне захоплення знімків (v1.1.0)**
   - 📸 Коли виявлена особа - автоматично робиться селфі
   - 💾 Знімок зберігається у внутрішній пам'ять
   - ☁️ Завантажується в Google Photos (синхронізація)
   - 🖼️ Доступ до всіх знімків через нову вкладку
   - 📊 Перегляд, видалення та керування знімками

### 3. **Система сповіщень (v1.0.1 покращено)**
   - Яскраве виконавче сповіщення
   - 🔊 Звукові сигнали з настройками
   - 📳 Вібрація з поліпшеною підтримкою
   - Повноекранне попередження про спробу доступу
   - 🎬 Анімовані алерти
   - Автоматична фіксація часу і дати інциденту

### 4. **Білий список контактів**
   - Додавайте дозволені контакти
   - Автоматично пропускаються сповіщення для них
   - Легко керуйте списком дозволених осіб
   - 🚀 DiffUtil для бистрого оновлення списку

### 5. **Історія спроб (v1.0.1 покращено)**
   - Детальна історія всіх спроб доступу
   - Час та дата кожної спроби
   - Легко видалити або переглянути записи
   - ⚡ Оптимізована робота з БД

### 6. **🆕 Менеджер захоплень (v1.1.0)**
   - Переглядайте всі захоплені селфі в сітці
   - Видаляйте небажані знімки
   - Завантажуйте їх в Google Photos
   - Моніторьте розмір сховища (МБ)
   - Масове управління знімками

### 7. **Налаштування (v1.1.0 розширено)**
   - Вмикання/вимикання сповіщень
   - Вібрація та звук
   - 🆕 Конфігурація захоплень знімків
   - Детальна інформація про додаток
   - 📊 Перегляд статистики


1. Підключіть пристрій (USB Debug) або запустіть емулятор
2. Натисніть Run ▶️ або Shift + F10
3. Виберіть цільовий пристрій

## 🐛 Вирішення проблем

### Проблема: Камера не активується
- Переконайтесь, що дозвіл на камеру надано
- Перевірте, що інша програма не використовує камеру
- Перезапустіть додаток
- Перезавантажте пристрій

### Проблема: Сповіщення не приходять
- Перевірте налаштування сповіщень в системі
- Переконайтесь, що додаток має дозвіл на сповіщення
- Вімкніть сповіщення у налаштуваннях додатку
- На Android 12+ надайте дозвіл на сповіщення

### Проблема: Знімки не зберігаються (🆕 v1.1.0)
- Перевірте дозволи WRITE_EXTERNAL_STORAGE
- Переконайтесь що вільна місце у пам'яті
- Перевірте логи: `adb logcat | grep CaptureUtils`
- Спробуйте вручну надати дозвіл:
  ```bash
  adb shell pm grant com.screenguard.protector android.permission.WRITE_EXTERNAL_STORAGE
  ```

### Проблема: Google Photos синхронізація не працює (🆕 v1.1.0)
- Перевірте інтернет підключення
- Переконайтесь що користувач авторизований
- Перевірте логи: `adb logcat | grep GooglePhotosManager`
- На даний момент - MOCK реалізація, потребує OAuth налаштування

## 📊 Архітектура

### Design Pattern: MVVM + Repository
- **Model**: Room Database, DataStore
- **View**: Activities, Adapters, Layouts
- **ViewModel**: PreferencesManager, Repositories

### Основні компоненти:
- **Service**: Фоновий сервіс для детекції осіб та захоплення знімків
- **Repository**: Абстракція доступу до даних
- **Database**: Room для локального зберігання
- **UI**: Material Design 3 інтерфейс
- 🆕 **CaptureUtils**: Обробка кадрів та збереження знімків
- 🆕 **GooglePhotosManager**: Синхронізація з Google Photos

> **Детальне описання**: [ARCHITECTURE.md](ARCHITECTURE.md)

## 🔄 Життєвий цикл

1. **Запуск**: Service стартує і активує CameraX
2. **Аналіз**: ML Kit постійно аналізує кадри
3. **Детекція**: Якщо виявлена особа - спрацьовує алерт
4. **🆕 Захоплення**: Автоматично робиться селфі-знімок
5. **Запис**: Інцидент та знімок зберігаються
6. **🆕 Синхронізація**: Знімок завантажується в Google Photos
7. **Сповіщення**: Користувач отримує сповіщення та алерт
8. **Зупинка**: Service завершується при деактивації

## 📈 Оптимізація (v1.1.0)

### Продуктивність
- ⚡ ImageAnalysis з STRATEGY_KEEP_ONLY_LATEST
- ⚡ ML Kit з PERFORMANCE_MODE_FAST
- ⚡ DiffUtil в адаптерах для бистрого оновлення
- 🆕 ⚡ Cooldown 5 сек між захопленнями (оптимізація батареї)
- ⚡ Executor для паралельної обробки камери

### Батарея
- 🔋 Foreground Service (не kill від системи)
- 🔋 3 секундний cooldown між алертами
- 🔋 🆕 5 секундний cooldown між захопленнями
- 🔋 Мінімальна обробка даних

### Пам'ять
- 💾 Правильна очистка ресурсів
- 💾 Закриття imageProxy у finally блоку
- 💾 Shutdown ExecutorService
- 🆕 💾 Асинхронне збереження знімків (не блокує UI)

## 🐛 Вирішення проблем

### Проблема: Камера не активується
- Переконайтесь, що дозвіл на камеру надано
- Перевірте, що інша програма не використовує камеру
- Перезапустіть додаток
- Перезавантажте пристрій

### Проблема: Сповіщення не приходять
- Перевірте налаштування сповіщень в системі
- Переконайтесь, що додаток має дозвіл на сповіщення
- Вімкніть сповіщення у налаштуваннях додатку
- На Android 12+ надайте дозвіл на сповіщення

### Проблема: Повільна детекція
- Це нормально при першому запуску (завантаження ML Kit модели)
- Детекція прискорюється після першого використання
- Збільште освітлення в кімнаті для точнішої детекції
- Очистіть RAM памяти на пристрої

> **Повна документація**: [INSTALLATION_GUIDE.md](INSTALLATION_GUIDE.md)

## 📊 Архітектура

### Design Pattern: MVVM + Repository
- **Model**: Room Database, DataStore
- **View**: Activities, Adapters, Layouts
- **ViewModel**: PreferencesManager, Repositories

### Основні компоненти:
- **Service**: Фоновий сервіс для детекції осіб
- **Repository**: Абстракція доступу до даних
- **Database**: Room для локального зберігання
- **UI**: Material Design 3 інтерфейс

> **Детальне описання**: [ARCHITECTURE.md](ARCHITECTURE.md)

## 🔄 Життєвий цикл

1. **Запуск**: Service стартує і активує CameraX
2. **Аналіз**: ML Kit постійно аналізує кадри
3. **Детекція**: Якщо виявлена особа - спрацьовує алерт
4. **Запис**: Інцидент зберігається в базі даних
5. **Сповіщення**: Користувач отримує сповіщення та алерт
6. **Зупинка**: Service завершується при деактивації
